# cowcow CLI

## Overview

cowcow is a powerful command-line interface designed for offline-first data capture and processing. Built with Rust for maximum performance and reliability, cowcow enables researchers, developers, and data scientists to collect, process, and synchronize data even in environments with limited or no internet connectivity.

## Key Features

### Offline-First Architecture
- **Local Storage**: SQLite database for reliable local data storage
- **Sync Capabilities**: Intelligent synchronization when connectivity is available
- **Conflict Resolution**: Automated conflict resolution for concurrent edits
- **Data Integrity**: Built-in validation and error recovery

### High Performance
- **Rust-Powered**: Lightning-fast processing with memory safety
- **Concurrent Processing**: Multi-threaded data processing
- **Minimal Dependencies**: Lightweight binary with no external dependencies
- **Cross-Platform**: Works on Linux, macOS, and Windows

### Developer-Friendly
- **Simple API**: Intuitive command-line interface
- **Extensible**: Plugin architecture for custom processors
- **Scriptable**: Easy integration with existing workflows
- **Rich Documentation**: Comprehensive guides and examples

## Architecture

```bash
git clone https://github.com/deep-ubuntu/cowcow

```

## Usage Examples

### Basic Data Capture
```bash
# Initialize a new project
cowcow init my-project

# Capture data from stdin
echo '{"name": "John", "age": 30}' | cowcow capture --format json

# Process a CSV file
cowcow process data.csv --transform normalize --output processed.json

# Sync with remote endpoint
cowcow sync --endpoint https://api.example.com/data
```

### Advanced Processing
```bash
# Chain multiple processors
cowcow process data.csv \
  --transform "filter:age>18" \
  --transform "group:by=country" \
  --transform "aggregate:sum=population" \
  --output results.json

# Real-time processing
cowcow watch /data/input \
  --on-change "process --transform normalize" \
  --sync-interval 5m
```

## Configuration

Create a `cowcow.toml` configuration file:

```toml
[project]
name = "research-data"
version = "1.0.0"

[storage]
database = "data.db"
backup_interval = "1h"

[sync]
endpoint = "https://api.research.org/sync"
auth_token = "${COWCOW_AUTH_TOKEN}"
batch_size = 1000

[processing]
workers = 4
memory_limit = "1GB"

[[processors]]
name = "normalize"
config = { remove_nulls = true, lowercase_keys = true }

[[processors]]
name = "validate"
config = { schema = "schema.json" }
```
